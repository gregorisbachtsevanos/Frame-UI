# Design System Tokens

Frame UI uses a comprehensive set of design tokens to ensure consistency across all components and your entire application.

## Overview

Design tokens are the building blocks of the Frame UI design system. They define colors, spacing, typography, shadows, and transitions used throughout the library.

```tsx
import { vars } from '@frame-ui/theme';

// Access design tokens
const color = vars.color.primary;      // #2563eb
const space = vars.space.lg;           // 16px
const radius = vars.radius.md;         // 12px
const shadow = vars.shadow.md;         // 0 2px 10px rgba(...)
```

## Typography

### Font Family

```tsx
vars.font.body   // 'Inter, sans-serif'
```

### Font Sizes

```tsx
vars.fontSize.xs   // 12px - Small labels, hints
vars.fontSize.sm   // 14px - Secondary text, captions
vars.fontSize.md   // 16px - Body text (default)
vars.fontSize.lg   // 18px - Subheadings
vars.fontSize.xl   // 20px - Headings
```

### Usage Examples

<div style={{ marginBottom: '32px' }}>

```tsx
import { vars } from '@frame-ui/theme';

// Small label
<span style={{ fontSize: vars.fontSize.xs }}>Helper text</span>

// Body text
<p style={{ fontSize: vars.fontSize.md }}>Paragraph content</p>

// Large heading
<h1 style={{ fontSize: vars.fontSize.xl }}>Main title</h1>
```

## Transitions

Frame UI defines two transition speeds:

```tsx
vars.transition.fast   // 150ms ease
vars.transition.normal // 250ms ease
```

### Usage Examples

```tsx
import { vars } from '@frame-ui/theme';

// Fast interaction feedback
style={{
  transition: `color ${vars.transition.fast}`,
}}

// Smooth layout changes
style={{
  transition: `all ${vars.transition.normal}`,
}}
```

## Dark Mode

All tokens automatically adjust when dark mode is enabled:

```tsx
import { darkTheme } from '@frame-ui/theme';

// Enable dark mode
document.documentElement.classList.add(darkTheme);

// In dark mode, tokens update automatically:
// text: #f1f5f9 (light)
// surface: #0f172a (dark)
// border: #334155 (darker)
```

## Best Practices

### 1. Always Use Tokens

```tsx
// ✅ Good - Using design tokens
const color = vars.color.primary;
const space = vars.space.lg;

// ❌ Avoid - Hardcoded values
const color = '#2563eb';
const space = '16px';
```

### 2. Consistent Spacing

```tsx
// ✅ Good - Consistent scale
padding: vars.space.lg;
margin: vars.space.md;
gap: vars.space.sm;

// ❌ Avoid - Mixed spacing
padding: '15px';
margin: '10px';
gap: '7px';
```

### 3. Semantic Color Usage

```tsx
// ✅ Good - Semantic naming
color: vars.color.text;
background: vars.color.surface;

// ❌ Avoid - Arbitrary colors
color: '#0f172a';
background: '#ffffff';
```

### 4. CSS-in-JS Usage

```tsx
// ✅ Good - Using inline styles
<div style={{
  padding: vars.space.lg,
  color: vars.color.text,
}}>
  Content
</div>

// ✅ Also good - Using CSS classes with Vanilla Extract
import * as styles from './Component.css';

// ❌ Avoid - Hardcoded CSS
<div style={{ padding: '16px', color: '#0f172a' }}>
  Content
</div>
```

## Customization

To customize design tokens for your project:

1. Fork the `@frame-ui/theme` package
2. Update the token values in `packages/theme/src/tokens.css.ts`
3. Update dark mode overrides in `packages/theme/src/dark.css.ts`
4. Rebuild and publish your custom version
